MovingAgentDemo Readme

 

Description

This sample demonstrates an agent that starts its execution on the host where it is loaded and then migrates to another host. We will launch two instances of EtherYatri.NET Windows Agent Host - WINAH and load the mobile agent in one of them. We’ll see that the agent moves from the host where it was loaded to the other host.

 

What you will learn…

·         Writing mobile agents that migrate from one host to another.

·         Passing initialization arguments to the agent’s OnCreation method through WINAH.

 

Requirements

·         Microsoft Visual Studio.NET

·         EtherYatri.NET Toolkit (available from http://www.geocities.com/siddharthuppal/EtherYatri.htm).

 

Concepts Involved

Remember that a moble-agent’s OnCreation method is called once during the entire lifetime of the agent while its Run method is called on every host where it starts executing.

 

On loading the agent into WINAH, first its OnCreation method would be called. Next, its Run method will be called. In the Run method we can perform the migration by invoking MoveTo passing the address of the destination host. But that’s not all. There’s a problem. When our agent has reached the destination host, its Run method will be called again by the EtherYatri.NET Framework. And in the Run method we have the MoveTo statement that requests the migration of the agent to the host which it has already reached. How do we avoid this problem?

 

The EtherYatri.NET Framework saves the values of the fields of the agent between transfers. We will use an integer field, called state to remember whether we have performed the migration. The EtherYatri.NET Framework at the destination host, after receiving the agent, would first call the agent’s OnArrival method and then its Run method. In the OnArrival method we will increment the value of the integer field – state. We will write the Run method such that we check the value of the integer field – state before invoking MoveTo. We call MoveTo only if the value of state is zero.
